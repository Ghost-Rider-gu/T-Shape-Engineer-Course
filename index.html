T-Shape Engineer

Hi everyone and welcome to the T-Shape course!

This is a breif introduction where I'll provide you with some information about that course. So, T-Shape is a course (mostly for Java developers) that includes two big parts - T-Shape Engineer articles and T-Shape Engineer project. Plus, this small web page which you can use as a navigation tool to get through the course.

In the first part of the T-Shape course (T-Shape Engineer articles) I would like to share my knowledge with you about backend and frontend. As I'm a Java developer the main language will be Java. We will talk about Java itself, computer sience, network programming, patterns, architecture, microservices, security. Regarding frontend there will be JavaScript and some of the frameworks such as VueJS and ReactJS, PWA (Progressive Web Apps), DevOps, and so on. I will try to organise all these stuff in the understandable and accessible form.

The second part is more like a phisical representation of the first part, we will create a project called "Mister Keeper". The whole process of creation of this project is based on knowledge shared in the first part, so step by step, article by article "Mister Keeper" will be creating from scratch. "Mister Keeper" is a web application where you can keep information about books, music, movies, games. Searching, rating, reviewing, sharing information with you friends and so on (it's kind of social network). All in one place. "Mister Keeper" has two parts: Backend part (the source code is here) and Frontend part (the source code is here)

The last but not least this "navigation" tool. You can consider it as a starting point to start your journey. I hope it will help you go through the articles and see the whole picture of the course. You can start from the current small description then take a look at the T-Shape Engineer schema - it's a diagram where depicts all topics that will be covered. Maybe I will extend it in near future with new interesting topics. The last thing it's T-Shape Engineer articles - also a diagram with links to the specific article on DZone.com portal. This diagram will be growing with every new article published on DZone.

So, basically that's all. I wish you all the best and take care!
